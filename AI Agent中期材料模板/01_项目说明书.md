# <项目名称>项目说明书

## 一、项目背景与问题定义

请说明本项目希望解决的具体问题。

建议包含：

1. 应用场景：该Agent面向什么用户或任务。
2. 当前痛点：现有的相关产品或方法存在哪些问题/现有流程中哪些环节低效、容易出错或需要自动化/哪些任务无法完成。
3. 项目目标：希望项目能够达成哪些效果或目标。
4. 边界说明：当前版本不解决哪些问题。

## 二、Demo 功能范围

请说明本阶段 Agent Demo 具体包含哪些功能，以及当前版本的能力边界。

建议包含：

1. **输入形式**：用户可以向 Agent 输入什么内容，例如自然语言需求、文件、数据、问题描述等。
2. **输出形式**：Agent 最终返回什么结果，例如文本回答、分析报告、推荐方案、代码、图表或操作记录等。
3. **核心功能**：当前 Demo 支持哪些主要功能模块。
4. **功能边界**：当前版本暂不支持哪些复杂任务或异常情况。

## 三、系统设计

请说明系统如何实现上一节所列出的 Demo 功能范围，包括整体架构、模块划分、执行流程和评估指标等。*如有基于搭载英特尔酷睿Ultra处理器的PC构建采用混合（云端+本地端）AI架构的Agent或Skill等符合附加分的情况，请在此部分详细说明。*

建议包含：

1. **系统架构**：简要描述系统的整体结构，可结合文字说明、流程图或架构图展示。
2. **模块说明**：各模块分别负责什么输入、处理逻辑和输出结果。
3. **执行流程**：用户请求进入系统后，系统如何完成处理并返回结果。
4. **评估指标**：从哪些方面观察 Demo 的运行效果，可结合项目特点选择定性或定量方式。另外需要针对评估使用的数据集规模做出相应说明。例如：
    1. **任务完成情况**：系统是否能完成典型任务。
    2. **工具调用正确性**：是否在合适步骤调用了正确工具或模块。
    3. **结果质量**：最终输出是否准确、完整、清晰。
    4. **运行稳定性**：执行过程中是否出现报错、中断或无效结果。
    5. **过程可检查性**：是否能查看关键步骤、中间结果和调用记录。
