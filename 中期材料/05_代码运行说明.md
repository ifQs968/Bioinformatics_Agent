# 代码运行说明

## 一、运行环境要求

| 环境 | 版本要求 | 说明 |
|------|----------|------|
| Python | >= 3.10 | 后端运行环境，推荐 3.11+ |
| Node.js | >= 18 | 前端构建与开发服务器 |
| pip | 最新版 | Python 包管理 |
| conda（可选） | 任意版本 | 推荐用于 Python 环境隔离 |

## 三、安装流程

### Step 1: 克隆仓库

```bash
git clone https://github.com/BeatGene/Bioinformatics_Agent.git
cd Bioinformatics_Agent
```

### Step 2: 创建 Python 环境并安装后端依赖

```bash
# 创建 conda 环境（推荐）
conda create -n BioAgent python=3.11 -y
conda activate BioAgent

# 安装 Python 依赖
pip install -r requirements.txt

# 如遇 urllib3 兼容性问题，升级 urllib3：
pip install --upgrade urllib3 requests
```

### Step 3: 配置环境变量

在项目根目录创建 `.env` 文件：

```bash
# DeepSeek API 配置
# 去 https://platform.deepseek.com 注册获取 API Key
DEEPSEEK_API_KEY=sk-your-api-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# PubMed 配置（可选，不填也能正常使用）
PUBMED_EMAIL=
PUBMED_API_KEY=

# 应用配置
APP_HOST=127.0.0.1
APP_PORT=7860
LOG_LEVEL=INFO
```

> **重要**：请将 `DEEPSEEK_API_KEY` 替换为你自己的 DeepSeek API Key。

### Step 4: 安装前端依赖

```bash
cd frontend
npm install
```

## 六、常见问题

### Q1: `ModuleNotFoundError: No module named 'urllib3.packages.six.moves'`

**原因**：conda 环境中 `urllib3` 版本过旧。

**解决**：

```bash
pip install --upgrade urllib3 requests
```

### Q2: `ValueError: 请在 .env 文件中设置有效的 DEEPSEEK_API_KEY`

**原因**：`.env` 文件中未填写有效的 API Key。

**解决**：在项目根目录创建或编辑 `.env`，填入你的 DeepSeek API Key。

### Q3: 前端访问后端 API 报 CORS 错误

**原因**：前端开发服务器（5173）通过 Vite 代理转发到后端（7860），需确保后端已启动。

**解决**：确认终端 1 中 `python backend/main.py` 正在运行。

### Q4: PubMed 检索返回 0 条结果

**原因**：检索式过于具体或中文关键词未正确翻译。

**解决**：系统已内置 LLM 翻译 + 宽泛关键词重试机制，通常可自动恢复。如持续失败，尝试更简洁的英文关键词。

### Q5: 前端构建 chunk 超过 500KB 警告

**原因**：KaTeX（数学公式渲染）字体文件较大。

**说明**：仅为警告，不影响功能。可通过代码分割优化，不影响开发使用。

## 七、技术栈

### 后端

- **Web 框架**：FastAPI + uvicorn
- **LLM**：DeepSeek API（兼容 OpenAI SDK 格式）
- **Agent 编排**：LangGraph（状态定义）+ 自研 Executor
- **文献检索**：pymed（PubMed API 封装）
- **PDF 解析**：MinerU + PyMuPDF（双引擎，已实现待接入）
- **流式传输**：SSE（Server-Sent Events）

### 前端

- **框架**：React 19 + TypeScript 6
- **构建**：Vite 8
- **样式**：Tailwind CSS v4
- **Markdown 渲染**：react-markdown + remark-gfm + rehype-katex
- **图标**：lucide-react
- **状态管理**：React useState/useEffect（无第三方库）
